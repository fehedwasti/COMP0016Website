Requirements

Introduction


Immersive Health is a healthcare service which aims to use technologies such as virtual reality to provide solutions to current healthcare problems. The aim of this project is to produce a library that uses a skeletal tracker which can be implemented into a virtual reality game that allows users to train physically by practicing a sport using VR-headsets.


In order to begin research and development of the library it is important to understand the requirements that need to be met for the client. For the first few weeks, our team setup weekly meetings with the client to demonstrate new ideas and solutions. The feedback recieved helped in determining exactly what the client expected from us and from that, we were able to identify the main project requirements.



MoSCoW Requirements



Must Have Should Have Could Have

Essential Features

- Implementation of skeletal tracking
- Features targeting physical activity and sports practice
- Feedback to help users improve their performance.

Preferred Features

- An easy to use interface
- Use of the XBox Kinect for motion tracking
- The Library should be easy to implement into software

Additional Features

- A multiplayer feature allowing friends to spar together
- Background Music
- A ranking system or leaderboard for highest scorers
- Multiple environments for boxing
- Allows practice for multiple sports



Personas


Software Developer

Our team is developing a library which can be reused by other developers if they need skeletal tracking in their applications.

Boxers

We will use the library we create to develop a virtual reality boxing game that will allow users to practice their boxing techniques and train their bodies in physical activities.

Patients

Some patients may need to exercise their muscle movements and bones as part of their treatment and the VR game can be an excellent tool in providing just that.
