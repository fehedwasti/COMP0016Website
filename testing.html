Testing

Unit Testing & UAT



Upon arriving at a stage where we had a working prototype, testing had become a vital part of further developing the game. As a result, we wrote several different unit tests to ensure all aspects of the skeletal tracking were behaving as expected. Before each stage of adding more features to the game, we would write more test scripts for the new features and rerun all tests after a change is implemented.

User Acceptance Testing is just as important as Unit testing. Every week the group would come together and test the game along with our Teaching Assistant or our Client. This was to make sure the users of the application found the game interesting and engaging.




Compatibility Testing



It's also important to understand the different hardwares that could be used with our skeletal tracking library and so once we had our working prototype, we used the Kinect sensor and tried playing the game on computers with different specifications. We noticed that the game was only compatible on computers running windows although laptops primarily running MacOS had the option to install Windows using Bootcamp and use the library.

However, the VR boxing game we developed using the library requires users to use the Windows Mixed Reality Headsets which are only compatible with computers originally built to run Windows. MacOS users can not make use of the boxing game. For the best experience, a computer with a high-end graphics card is recommended.
