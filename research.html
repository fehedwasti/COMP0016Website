Research

Existing Solutions

As of today, most of the solutions are virtual reality games that allow users to play all sorts of sports including baseball, cricket, hockey and many more. However, despite these games being very fun and immersive, they are not very active as the player normally stands in one spot and only swing their arms.




Understanding the User


To start off our project, despite understanding the requirements of the client, we needed to know what a user might want. As part of our HCI Design stage, we came up with several sketches and ideas and for our research, we used these sketeches and asked the personas which idea they preferred the most.




Related Technologies


Unreal Engine

This fully open source framework allows development of applications with top tier graphics has a large suite of tools. With a large community it shouldn't be too difficult to fix issues should they arise during development. This framework also has a middleware plugin that allows developers to use the Kinect sensor.

Unity

Unity is another open source framework with a larger suite of tools and features allowing more creativity during development. The community is very large as the framework has been available for many years and it is much simpler to use the Kinect SDK with this framework

CryENGINE

CryEngine is one of the most powerful frameworks allowing developers to produce high quality smooth and stable games and applications. The source code for the framework was released much later than the other 2 frameworks. Furthermore, it is more complex and difficult to understand and use.


Final Decision

Our final decision was to use Unity, simply because it provides plenty of resources and tutorials, is very user friendly and shouldn't cause too many problems as it allows easy usage of the Kinect SDK. CryENGINE seemed too complicated and UnrealEngine had more or less the same features of Unity.
